# 📐 Chapter 1: Plan & Architect

> Keep it simple, or no one cares.

Before writing a single line of code, you need to map your process and plan your product. Here are the tools shown in the session.

---

## 🛠️ Tools

### FigJam (Process Mapping)
Used to map out workflows like the **Digital Loan Flow** shown in the slide.
- 🔗 [figjam.com](https://www.figma.com/figjam/)
- Use it for: User journey maps, system architecture diagrams, sprint planning boards
- Free for teams getting started

### Notion (Product Roadmap)
Used to build the full product roadmap for **Somadhan Legal** — tracking modules, database entities, app surfaces, and team responsibilities.
- 🔗 [notion.so](https://notion.so)
- Use it for: Product roadmaps, project trackers, data rooms, HR docs
- Free for individuals

### Miro (Alternative to FigJam)
- 🔗 [miro.com](https://miro.com)

---

## 📋 Prompt to Try in Notion AI
